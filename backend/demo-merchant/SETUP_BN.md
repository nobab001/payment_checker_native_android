# ডেমো মার্চেন্ট — সম্পূর্ণ সেটআপ গাইড (বাংলায়)

> এই গাইড এমনভাবে লেখা হয়েছে যেন আপনি নিজেই একটা ওয়েবসাইট বানিয়েছেন এবং PayCheck থেকে পেমেন্ট গেটওয়ে একাউন্ট নিয়েছেন — এখন দুটো কীভাবে জোড়া লাগাবেন।

---

## ১. ডেমো মার্চেন্ট আসলে কী?

এটা **আসল ই-কমার্স সাইট না**। এটা একটা **অনুশীলনের দোকান** — যেন PayCheck পেমেন্ট সিস্টেম ঠিকমতো কাজ করছে কিনা সেটা পরীক্ষা করা যায়।

**সহজ ভাষায়:**
- আপনার দোকান = ডেমো মার্চেন্ট ওয়েবসাইট
- পেমেন্ট মেশিন = PayCheck (অ্যাপ + চেকআউট পেজ)
- দোকান পেমেন্ট মেশিন বানায় না — শুধু ব্যবহার করে

---

## ২. কোন টেকনোলজি দিয়ে বানানো?

| অংশ | কী ব্যবহার করা হয়েছে |
|-----|----------------------|
| সার্ভার (ব্যাকএন্ড) | **Node.js** + **Express** |
| ডাটাবেস | **MySQL** (PayCheck-এরই ডাটাবেস শেয়ার করে) |
| ডাটা অ্যাক্সেস | **Prisma** (ডাটাবেসের সাথে কথা বলার টুল) |
| লগইন সুরক্ষা | **JWT** টোকেন + **bcrypt** পাসওয়ার্ড |
| ফ্রন্টএন্ড (ওয়েব পেজ) | **HTML + CSS + JavaScript** (কোনো React/Vue নেই — সরল ও দ্রুত) |
| পেমেন্ট | PayCheck-এর **আগে থেকে বানানো API** — আমরা নতুন পেমেন্ট লজিক বানাইনি |
| ছবি আপলোড | **Multer** + **Sharp** (ছবি ছোট করে সেভ করে) |

**ফোল্ডার:** `backend/demo-merchant/`

**ওয়েব ঠিকানা:** `http://আপনার-কম্পিউটার-আইপি:3000/demo-merchant/`

## ৩. নতুন ফিচার — এডমিন সিস্টেম

### সাধারণ ইউজার (Customer)
- অ্যাকাউন্ট খুলতে পারে, লগইন করতে পারে
- ওয়ালেট দেখে, ব্যালেন্স যোগ করতে পারে
- প্রোডাক্ট দেখে **Buy Now** চাপতে পারে
- প্রোডাক্ট **শুধু দেখতে পায়** — এডিট করতে পারে না

### এডমিন ইউজার
- উপরে **「এডমিন মোড」** বাটন দেখবে
- বাটন চাপলে **এডমিন ড্যাশবোর্ড**-এ যাবে
- সেখানে করতে পারবে:
  - প্রোডাক্ট **যোগ / এডিট / ডিলিট**
  - প্রোডাক্টের **ছবি আপলোড**
  - অন্য ইউজারকে **এডমিন পারমিশন দেওয়া বা নেওয়া**

**গুরুত্বপূর্ণ:** যে প্রথম ইউজার রেজিস্টার করবে, সে অটো **এডমিন** হবে। পরের সবাই সাধারণ ইউজার।

## ৪. ধাপে ধাপে — PayCheck অ্যাপে ওয়েবসাইট যোগ করা

### ধাপ ১: ব্যাকএন্ড সার্ভার চালু করুন

```bash
cd backend
npm start
```

আপনার কম্পিউটারের **লোকাল IP** বের করুন (যেমন `192.168.0.105`)।  
ফোন ও PC **একই Wi-Fi**-তে থাকতে হবে।

---

### ধাপ ২: PayCheck অ্যাপ খুলুন

1. অ্যাপে **লগইন** করুন
2. নিচে বা মেনুতে **「API」** বা **「API ইন্টিগ্রেশন」** ট্যাবে যান
3. ডান পাশে **「+」** (প্লাস) বাটন চাপুন — নতুন ওয়েবসাইট যোগ করতে

---

### ধাপ ৩: ওয়েবসাইটের তথ্য দিন

| ফিল্ড | কী লিখবেন | উদাহরণ |
|-------|-----------|--------|
| **ডোমেইন** | ডেমো সাইটের ঠিকানা | `http://192.168.0.105:3000/demo-merchant` |
| **ওয়েবসাইটের নাম** (ঐচ্ছিক) | যেকোনো নাম | `Demo Merchant Shop` |

**সেভ** করুন।

---

### ধাপ ৪: API Key ও Secret সংরক্ষণ করুন

ওয়েবসাইট তৈরি হলে একবারই দেখাবে:

| নাম | দেখতে কেমন | কী করবেন |
|-----|-----------|----------|
| **API Key** | `pk_` দিয়ে শুরু | কপি করে রাখুন |
| **API Secret** | `sk_` দিয়ে শুরু | কপি করে রাখুন — **আবার দেখাবে না!** |

> Secret হারিয়ে গেলে অ্যাপে ওয়েবসাইট সেটিংসে **「Secret Rotate」** দিয়ে নতুন বানাতে হবে।

---

### ধাপ ৫: Callback URL সেট করুন

ওয়েবসাইটের **সেটিংস**-এ গিয়ে এই URL দিন:

```
http://192.168.0.105:3000/demo-merchant/api/webhooks/paychek
```

(`192.168.0.105` এর জায়গায় আপনার আসল IP দিন)

**Callback URL মানে কী?**  
কাস্টমার পেমেন্ট শেষ করলে PayCheck **আপনার দোকানকে ফোন করে বলে** — 「পেমেন্ট হয়ে গেছে, ওয়ালেট আপডেট করো।」

---

## ৫. API Key ও Secret কোথায় বসাবেন?

ফাইল: **`backend/.env`** (backend ফোল্ডারের ভিতরে)

```env
# PayCheck থেকে পাওয়া কীগুলো
DEMO_MERCHANT_PAYCHEK_API_KEY=pk_xxxxxxxxxxxxxxxx
DEMO_MERCHANT_PAYCHEK_API_SECRET=sk_xxxxxxxxxxxxxxxx

# আপনার কম্পিউটারের IP (ফোন/ইন্টারনেট থেকে পৌঁছাতে হবে)
DEMO_MERCHANT_PUBLIC_URL=http://192.168.0.105:3000

# (ঐচ্ছিক) সার্ভার-টু-সার্ভার PayCheck API — ডিফল্ট http://127.0.0.1:PORT
# PUBLIC_URL ব্রাউজার/চেকআউটের জন্য; API কল সবসময় loopback দিয়ে যায় (Windows timeout এড়াতে)
# DEMO_MERCHANT_PAYCHEK_API_URL=http://127.0.0.1:3000

# (ঐচ্ছিক) লগইন টোকেন সুরক্ষা
DEMO_MERCHANT_JWT_SECRET=আপনার-গোপন-শব্দ-যেকোনো-লম্বা-টেক্সট
```

**কোডের ভিতরে হাতে লিখতে হবে না** — শুধু `.env` ফাইলে বসান।  
কোড `backend/demo-merchant/config.js` থেকে এই মানগুলো পড়ে।

`.env` সেভ করার পর **সার্ভার রিস্টার্ট** করুন:

```bash
# Ctrl+C দিয়ে বন্ধ করে আবার
npm start
```

## ৬. ডাটাবেস সেটআপ (প্রথমবার)

```bash
cd backend
npm run demo-merchant:migrate
npm run demo-merchant:migrate-admin
npm run demo-merchant:seed
```

| কমান্ড | কাজ |
|--------|-----|
| `migrate` | টেবিল বানায় (ইউজার, ওয়ালেট, অর্ডার...) |
| `migrate-admin` | এডমিন `role` কলাম যোগ করে |
| `seed` | ৬টা ডেমো প্রোডাক্ট যোগ করে |

## ৭. পুরো পেমেন্ট ফ্লো — একটা গল্পের মতো

```
কাস্টমার                    ডেমো মার্চেন্ট              PayCheck
   |                              |                         |
   |--- Buy Now চাপে ----------->|                         |
   |                              |--- pay/init API ------->|
   |                              |<-- checkout URL --------|
   |<-- চেকআউট পেজে যায় --------|                         |
   |                              |                         |
   |--- bKash/Nagad দিয়ে পে ---->|------------------------>|
   |                              |                         |
   |                              |<-- webhook (পেমেন্ট OK)-|
   |                              |   (ওয়ালেট/অর্ডার আপডেট)|
   |<-- সাকসেস পেজ --------------|                         |
```

1. কাস্টমার **Buy Now** বা **Add Balance** চাপে
2. ডেমো মার্চেন্ট PayCheck-কে বলে: 「এই পরিমাণের পেমেন্ট চাই」
3. PayCheck **চেকআউট পেজ** খোলে (`checkout.html` — আগে থেকেই আছে, আমরা বদলাইনি)
4. কাস্টমার পেমেন্ট করে
5. PayCheck **webhook** পাঠায় → ডেমো মার্চেন্ট ওয়ালেট/অর্ডার আপডেট করে

## ৮. কোডে কোথায় কী আছে? (সংক্ষেপে)

```
backend/demo-merchant/
├── config.js          ← .env থেকে API Key/Secret পড়ে
├── index.js           ← ওয়েবসাইট মাউন্ট করে
├── services/
│   ├── paychek-client.js   ← শুধু PayCheck API কল করে
│   ├── webhook-service.js  ← পেমেন্ট সফল হলে ওয়ালেট বাড়ায়
│   └── admin-service.js    ← প্রোডাক্ট/ইউজার ম্যানেজ
├── routes/            ← API ঠিকানা (URL)
└── public/            ← ব্রাউজারে যা দেখেন (HTML/JS/CSS)
```

**আমরা যা করিনি:**
- `backend/payment/` স্পর্শ করিনি
- চেকআউট পেজ বদলাইনি
- নতুন পেমেন্ট লজিক বানাইনি

## ১০. সমস্যা হলে

| সমস্যা | সমাধান |
|--------|--------|
| Checkout খোলে না | `.env`-এ API Key/Secret ঠিক আছে কিনা দেখুন, সার্ভার রিস্টার্ট |
| পেমেন্ট হয়েছে কিন্তু ব্যালেন্স বাড়েনি | Callback URL সঠিক IP দিয়ে সেট করুন |
| Secret ভুলে গেছেন | PayCheck অ্যাপে Secret Rotate করুন, `.env` আপডেট করুন |
| এডমিন বাটন দেখা যায় না | `migrate-admin` চালান, বা অন্য এডমিন থেকে Admin দিন |
