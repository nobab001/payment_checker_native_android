# Output Style Rule — Bengali Summary

**Rule:** কাজ শেষ হলে প্রতিবার বাংলায় আউটপুট দিতে হবে — কীভাবে কী করলাম, এবং এরপর কীভাবে কী হবে (পরবর্তী ধাপ / UI-তে কেমন দেখাবে / কোন ফাইলে কী বদলাল)।

**ফরম্যাট:**

1. **কী করা হলো (সংক্ষেপে):** — প্রতিটি গুরুত্বপূর্ণ পরিবর্তন বাংলায় ১-২ লাইনে।
2. **প্রভাবিত ফাইল:** — পূর্ণ পাথ সহ।
3. **এখন কেমন দেখাবে (UI-তে):** — ইউজার কী দেখবে, কোথায় দেখবে, কোনো ডায়ালগ চলে গেলে / ব্যানার এলে তা উল্লেখ।
4. **পরবর্তী ধাপ (ঐচ্ছিক):** — একটা ছোট concrete পরামর্শ, প্রশ্ন আকারে না।

**নিয়ম:**
- সবসময় বাংলায় লিখতে হবে (ইউজারের ভাষা)।
- টেকনিক্যাল টার্ম (file name, class name, API) হুবহু রাখতে হবে।
- প্রশ্ন দিয়ে শেষ করা যাবে না — স্টেটমেন্ট আকারে concrete পরামর্শ দিতে হবে।
